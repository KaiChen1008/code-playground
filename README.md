# code-playground

**code-playground** is a lightweight, self-hosted code execution platform (similar to ideone.com or Go Playground). It allows users to write, format, and execute snippets across multiple programming languages in isolated, secure environments.

## ✨ Features

- **Multi-Language Support**: Run Go, Python, JavaScript, Rust, and C++.
- **Isolated Execution**: Every execution runs in a fresh Docker container via `testcontainers-go`.
- **Modern UI**: Instagram-inspired minimalist theme with a custom Monaco editor.
- **Code Formatting**: Native support for formatting (e.g., `gofmt` for Go).
- **Shareable Snippets**: Unique URL generation (e.g., `localhost:8091/{id}`) for sharing code.
- **Persistent Storage**: Lightweight, file-based storage in `./data` (no heavy database required).
- **Security & Stability**: Built-in rate limiting and resource management.

## 🚀 Tech Stack

- **Backend**: Go (Gin Gonic)
- **API Strategy**: OpenAPI/Swagger (Spec-first development)
- **Frontend**: Vanilla JS/CSS + Microsoft Monaco Editor
- **Environment Management**: Docker & Testcontainers
- **Configuration**: Viper (YAML)
- **Logging**: Logrus

## 🏗 Project Architecture

The project follows the **Clean Architecture** pattern to ensure modularity and high testability:

- **Delivery**: HTTP Handlers and routing (Gin).
- **UseCase**: Core business logic and orchestration.
- **Domain**: Business entities (generated by `go-swagger`) and interfaces.
- **Repository**: Data access layer (File-system implementation).
- **Runner**: Containerized execution engine.

## 🛠 Installation & Setup

### Prerequisites

- Go 1.25+
- Docker & Docker Compose
- `go-swagger` (optional, for regenerating API models)

### Local Development

1. **Clone the repository**:
   ```bash
   git clone https://github.com/linkaichen/code-playground.git
   cd code-playground
   ```

2. **Run the server**:
   ```bash
   make run
   ```
   The API will be available at `http://localhost:8091`.

3. **Run Tests**:
   ```bash
   make test
   ```

### Docker Deployment

To spin up the entire stack (Server + Code Execution Runtimes):

```bash
make up
```

This will build the server image and pull necessary language runtime images (Go, Python, Node.js, etc.) as defined in `config.yaml`.

## 📄 API Documentation

The API is documented using Swagger. You can find the specification in `api/swagger.yaml`.

- `POST /api/v1/run`: Execute a code snippet.
- `POST /api/v1/format`: Format a code snippet.
- `GET /api/v1/snippet/:id`: Retrieve a stored snippet.
- `DELETE /api/v1/snippet/:id`: Remove a snippet.
- `GET /api/v1/languages`: List supported languages and versions.
