# SEO Strategy: PolyRun (Code Playground)

## 1. Executive Summary
This document outlines the strategy to improve the search engine visibility of **PolyRun**, a self-hosted, multi-language code playground. The goal is to rank for relevant keywords related to online code execution, particularly for Go, Python, and JavaScript.

## 2. Competitor Analysis

### Major Competitors
1.  **Replit:**
    *   **Strengths:** Dominant brand, full IDE features, community features, strong domain authority.
    *   **Weaknesses:** Can be slow, complex for simple snippets, aggressive monetization.
    *   **Opportunity:** Target users looking for a *faster, simpler, and free* alternative.

2.  **CodePen / JSFiddle:**
    *   **Strengths:** Huge community for frontend (HTML/CSS/JS).
    *   **Weaknesses:** Limited backend support (mostly frontend focused).
    *   **Opportunity:** Emphasize backend language support (Go, Python) which these tools lack or treat as secondary.

3.  **Go Playground (`play.golang.org`):**
    *   **Strengths:** Official tool, extremely fast, trusted.
    *   **Weaknesses:** Only supports Go, strict limits, no persistence (historically).
    *   **Opportunity:** Offer persistence (saving snippets) and multi-language support in one place.

4.  **Ideone:**
    *   **Strengths:** Supports many languages.
    *   **Weaknesses:** Dated UI, ads.
    *   **Opportunity:** Win on "Modern UI" and "Clean Experience" (no ads).

## 3. Keyword Strategy

### Primary Keywords (Head Terms)
*   `online code runner`
*   `code playground`
*   `run python online`
*   `go playground`
*   `javascript online compiler`

### Secondary Keywords (Long-Tail)
*   `self-hosted code execution`
*   `minimalist code editor online`
*   `secure code runner docker`
*   `polyrun code playground`
*   `alternative to replit for python`

### Targeted Phrases
*   "Run Go, Python, and JavaScript online instantly."
*   "Secure, isolated code execution environment."
*   "Share code snippets easily."

## 4. Technical SEO Checklist

-   [x] **Title Tags:** Unique and descriptive (e.g., "PolyRun - Online Code Runner").
-   [x] **Meta Descriptions:** Compelling summary for CTR.
-   [x] **Canonical URLs:** Prevent duplicate content issues (`https://polyrun.kaichenl.com`).
-   [x] **Structured Data:** `SoftwareApplication` schema to help Google understand the app.
-   [x] **Mobile Friendliness:** Ensure UI works on mobile (Viewport meta tag).
-   [ ] **Sitemap:** (Future) Generate sitemap for saved snippets if they become public.
-   [ ] **Performance:** (Ongoing) Keep `LCP` (Largest Contentful Paint) under 2.5s.

## 5. Content Recommendations (Future)
*   **"How-to" Guides:** Create a blog or documentation section with articles like "How to test Go concurrency online" or "Python data structures playground".
*   **Snippet Library:** Curate a list of "Featured Snippets" (e.g., "Bubble Sort in Go", "Async/Await in JS") that serve as landing pages for specific queries.
*   **Changelog:** Publicly document updates to signal activity to search engines.
